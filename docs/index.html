// PLATFORM FEATURES — COMPLETE PACKAGE

Everything a web team needs.
Nothing you don't.

Every feature below ships in the complete Skr8tr platform — orchestration, ingress, service discovery, PQC auth, compliance, and autoscaling as one sovereign stack. No assembling a toolchain. No Helm charts. No vendor add-ons.

🔐

Post-Quantum Auth

Every mutating command is ML-DSA-65 signed (CRYSTALS-Dilithium Level 3). Your signing key never touches the wire. Replay protection built in.

🔄

Rolling Updates

Launch new replicas, wait for settle, drain old ones. One command. No readiness probe YAML, no PodDisruptionBudget objects, no deployment strategy config required.

🌐

HTTP Ingress

Built-in reverse proxy with longest-prefix route matching. Dynamic backend resolution via the Tower on every request. No nginx, no Envoy config.

🧵

Masterless Mesh

No etcd, no Raft, no leader election. Every node is a sovereign peer. The Conductor coordinates lightweight heartbeats across the fleet. Add nodes by running the daemon — zero reconfiguration.

🔍

Service Discovery

Services register automatically on launch, deregister on kill. Round-robin across replicas. The ingress resolves live service addresses on every request — no stale routing tables, no manual config updates.

❤️

Health Checks

HTTP GET probes per workload. Configurable interval, timeout, retry count. Failed replicas are killed and relaunched automatically.

📜

Log Streaming

Per-process log buffer for every running service. Query from anywhere — the platform routes the request to the correct node automatically. No SSH. No kubectl exec. No log aggregation setup required.

⚡

Bare Metal Speed

No container runtime. No image registry. No pull latency. Your binary starts in milliseconds — from deploy command to serving traffic in under two seconds. Zero abstraction overhead on the critical path.

🔒

TLS/HTTPS Ingress

The built-in ingress terminates TLS natively. Backend connections stay plain — no re-encryption, no nginx TLS blocks, no ALB in front. HTTPS just works.

📊

Prometheus Metrics

Every node exposes a Prometheus metrics endpoint — process count, CPU%, free RAM, health failure counts. Scrape with any standard Prometheus stack. Zero config, no metrics-server required.

🧮

cgroups v2 Resource Limits

Declare memory and CPU limits per workload. Hard enforcement via cgroups v2 — every process stays within its allocated slice. No escape hatch, no resource contention between services.

⛓️

Secret Injection

Secrets are injected directly into the process at launch — never written to logs, never sent over the wire, never visible to other processes on the same host. Declare once, secure forever.

💾

Persistent Volumes

Mount host paths as environment variables for any workload. The directory is auto-created if missing. No PersistentVolumeClaim, no StorageClass, no provisioner setup.

♻️

Restart Policy

Three policies: always restart on any exit, restart only on failure, or never restart and rebalance cross-node. Placement state is always accurate — the platform tracks every relaunch.

⏳

Graceful Drain

Configure a drain window per service. On shutdown, the platform signals gracefully and waits the configured window before force-killing — giving processes time to flush queues, close DB connections, and finish in-flight requests.

🖥️

WASM + Full VM Workloads

Run native binaries, WASM modules, or full QEMU/Firecracker microVMs — all on the same node fleet. Mix workload types freely. One platform, one control plane, one ingress.

📡

Remote Exec & Dead Replica Detection

Run a shell command against any running workload — the platform routes it to the correct node and returns stdout. Dead replicas are detected and replaced instantly, not after a polling cycle.

💾

Persistent Conductor State

The Conductor persists workload state continuously. If it restarts, it replays state and re-establishes desired placement automatically — zero operator intervention, no lost assignments.

🧠

Autonomous Self-Healing Mesh

Dead replicas are detected and replaced instantly — no polling delay. Failed health probes kill and relaunch in-place. Autoscale triggers fire on CPU, memory, or custom Prometheus metrics without operator input. Restart policies enforce recovery across node reboots. The cluster heals itself — continuously, without a human in the loop.

// THE PLATFORM

The complete platform.
Not a toolchain. Not a kit.

Skr8tr ships as one complete proprietary platform — orchestration, ingress, service discovery, PQC auth, RBAC, audit chain, SSO, multi-tenancy, and autoscaling all in one licensed stack. Nothing to assemble. Nothing to bolt on. One vendor. One support line.

🔑 RBAC & Namespaces

Post-quantum signing keys per team. Teams are scoped to namespaces with fine-grained command ACLs: read / deploy / rollout / admin. One cluster, complete team isolation. No shared credentials, no lateral movement.

✓ INCLUDED IN PLATFORM

📋 Cryptographic Audit Ledger

SHA-256 chained ledger records every deploy, rollout, exec, and auth event. AES-256-GCM at-rest encryption. Tamper-evident by design — any modification breaks the chain. Full chain verification in seconds.

✓ INCLUDED  •  HIPAA • HITRUST • PCI DSS • NIST 800-53 • SOC 2

📡 RFC 5424 Syslog / SIEM Export

Every audit event forwarded in real time to your SIEM over TLS — Splunk, QRadar, Microsoft Sentinel, Datadog, Graylog, or any RFC 5424-compliant receiver. Covers HIPAA § 164.312(b) centralised log collection and PCI DSS 10.5.3.

✓ INCLUDED IN PLATFORM

🔐 SSO / OIDC Identity Bridge

Federate operator identity with Okta, Google Workspace, or any OIDC-compliant provider. Your IdP validates identity — Skr8tr maps group claims directly to RBAC namespaces. No separate user database. No password management.

✓ INCLUDED IN PLATFORM

🏛️ Multi-Tenant Conductor

Namespace isolation so team A cannot see or touch team B's workloads. Per-namespace resource quotas: max replicas, max RAM, max nodes, max submit rate. Real-time enforcement — over-quota SUBMIT is rejected before it touches a node.

✓ INCLUDED IN PLATFORM

📈 Auto-Scaling

CPU, memory, queue depth, and arbitrary Prometheus metric triggers. Cron-based predictive scaling for known traffic patterns. AND/OR composite rules. Min/max replica bounds enforced at the Conductor. No metrics-server, no HPA objects.

✓ INCLUDED IN PLATFORM

⚡ HTTP/2 Ingress

Full HTTP/2 with ALPN negotiation — multiplexed streams, header compression, gRPC proxying. Native TLS termination with automatic Let's Encrypt certificate management. Backends stay plain — no re-encryption overhead.
