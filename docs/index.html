// WHY SKR8TR

Kubernetes is an infrastructure problem.
Web teams have a shipping problem.

You know how to build services. You shouldn't need to become a platform engineer to run them. k8s was built for Google's scale in 2014. Skr8tr was built for web teams who have outgrown a single server but aren't Google. Three binaries. Any Linux host. Up in five minutes.

✗  Kubernetes

✗ 600+ MB control plane (etcd + apiserver + scheduler + controller-manager + ...)
✗ kubeconfig bearer tokens — base64-encoded secrets, effectively plaintext
✗ YAML + CRDs + Helm charts + kustomize layers
✗ Docker / containerd / runc / CNI — four abstraction layers to run a binary
✗ nginx-ingress as a separate Helm chart just to route HTTP
✗ Rollouts require readinessProbes, PodDisruptionBudgets, DeploymentStrategies
✗ Leader election, Raft consensus, SPOF etcd cluster
✗ No built-in audit trail — requires separate SIEM, Falco, or audit webhook plumbing
✗ Bearer tokens are base64 secrets — kubeconfig breaches grant full cluster access

✓  Skr8tr

✓ 5 MB total — three C23 binaries + a Rust CLI
✓ ML-DSA-65 post-quantum signatures — key never leaves your machine
✓ .skr8tr manifest — 6 lines to deploy an app
✓ No Docker. Fork+exec. Any binary, any language.
✓ Built-in HTTP ingress — longest-prefix routing, Tower discovery
✓ skr8tr rollout app.skr8tr — one command, zero downtime
✓ Masterless mesh — no etcd, no leader, no SPOF
✓ Prometheus metrics on every node (port 9100) — no metrics-server required
✓ cgroups v2 resource limits built in — memory.max + cpu.max per workload
✓ WASM + full VM (QEMU / Firecracker) alongside binary workloads on the same cluster
✓ SHA-256 audit chain + AES-256-GCM + SIEM syslog — HIPAA/HITRUST/PCI DSS built in

// PLATFORM FEATURES — COMPLETE PACKAGE

Everything a web team needs.
Nothing you don't.

Every feature below ships in the complete Skr8tr platform — orchestration, ingress, service discovery, PQC auth, compliance, and autoscaling as one sovereign stack. No assembling a toolchain. No Helm charts. No vendor add-ons.

🔐

Post-Quantum Auth

Every mutating command is ML-DSA-65 signed (CRYSTALS-Dilithium Level 3). Your signing key never touches the wire. Replay protection built in.

🔄

Rolling Updates

skr8tr rollout app.skr8tr — launch new replicas, wait for settle, drain old ones. One command. No readiness probe YAML required.

🌐

HTTP Ingress

Built-in reverse proxy with longest-prefix route matching. Dynamic backend resolution via the Tower on every request. No nginx, no Envoy config.

🧵

Masterless Mesh

No etcd, no Raft, no leader election. Every node is a sovereign peer. The Conductor schedules via UDP heartbeats. Add nodes by running the binary.

🔍

Service Discovery

Built-in Tower registry. Services register automatically on launch, deregister on kill. Round-robin across replicas. Query with skr8tr lookup.

❤️

Health Checks

HTTP GET probes per workload. Configurable interval, timeout, retry count. Failed replicas are killed and relaunched automatically.

📜

Log Streaming

Per-process ring buffer (last 200 lines). Query with skr8tr logs app. Auto-resolved to the correct node — no SSH required.

⚡

Bare Metal Speed

No container runtime. No OCI image pull. Fork + exec. Your binary starts in milliseconds, not seconds. Zero copy overhead on the data path.

🔒

TLS/HTTPS Ingress

The built-in ingress terminates TLS natively — pass --tls-cert and --tls-key to flip from HTTP to HTTPS. Backend connections stay plain. No Certbot config, no nginx TLS blocks.

📊

Prometheus Metrics

Every node exposes /metrics on port 9100 in Prometheus text format — process count, CPU%, free RAM, health failure counts. Scrape with any standard Prometheus stack. Zero config.

🧮

cgroups v2 Resource Limits

Declare ram and cpu in your manifest. The node writes memory.max and cpu.max into the workload's cgroup at launch. Hard enforcement, no escape hatch.

⛓️

Secret Injection

Declare secrets in a secret {} block. They are injected directly into the child process env after fork — never included in UDP wire commands, never written to logs, never visible to other processes.

💾

Persistent Volumes

Mount host paths as workload env vars via volume {}. The directory is auto-created if missing. The process sees DATA_DIR=/data/myapp. No PersistentVolumeClaim, no StorageClass.

♻️

Restart Policy

restart always — node relaunches locally on any exit. restart on-failure — only on non-zero. restart never — Conductor rebalances cross-node. All three send DIED/RELAUNCHED to keep placement accurate.

⏳

Graceful Drain

Set drain 30s in your manifest. On KILL, the node sends SIGTERM and waits the configured window before SIGKILL — giving your process time to flush queues, close DB connections, finish in-flight requests.

🖥️

WASM + Full VM Workloads

Declare type wasm to run a WASM module via wasmtime. Declare type vm with a vm {} block to launch a QEMU or Firecracker microVM. Binary, WASM, and VM workloads coexist on the same node fleet.

📡

Remote Exec & Dead Replica Detection

skr8tr exec myapp "ls -la /data" — the Conductor routes the shell command to the node hosting the app and returns stdout. Dead replicas are detected via instant SIGCHLD → DIED broadcast — no 15-second expiry wait.

💾

Persistent Conductor State

The Conductor writes its workload table to disk on every SUBMIT/EVICT. If the Conductor restarts, it replays the state file and re-establishes desired placement — zero operator intervention.

🧠

Autonomous Self-Healing Mesh

Dead replicas are detected instantly via SIGCHLD → DIED broadcast — no polling delay. Failed health probes kill and relaunch in-place. CPU autoscale triggers scale-up/down without operator input. Restart policies enforce recovery across node reboots. The cluster heals itself — continuously, without a human in the loop.

// MANIFEST FORMAT

Six lines to deploy. Not sixty.

The .skr8tr manifest is Skr8tr's sovereign alternative to Kubernetes YAML. Human-readable. No anchors. No CRDs.

api-server.skr8tr — basic service

app api-server exec /usr/local/bin/myapi args --port 8080 --db postgres.internal port 8080 replicas 3 restart on-failure drain 30s ram 536870912 # 512 MB cgroups limit cpu 2 health { check GET /healthz 200 interval 10s retries 3 }

with secrets, volumes, and scaling

app api-server exec /usr/local/bin/myapi port 8080 replicas 2 secret { DB_PASSWORD hunter2 API_KEY sk-abc123 } volume { DATA_DIR /data/myapp } scale { min 1 max 8 cpu-above 80 cpu-below 20 }

wasm workload

app data-processor type wasm exec /opt/modules/processor.wasm replicas 2 restart always volume { INPUT_DIR /data/input OUTPUT_DIR /data/output }

firecracker microVM

app isolated-worker type vm restart on-failure vm { hypervisor firecracker kernel /opt/vmlinux rootfs /opt/worker.ext4 vcpus 2 memory 512 net user }

// HOW IT WORKS

Write a manifest.
Deploy in seconds.

Skr8tr ships as pre-built binaries for your target platform. You get the Conductor, two worker node daemons, the CLI, and the ingress — everything configured, licensed, and ready. Here is what day one looks like.

1

You receive pre-built, signed binaries for your platform

No compiler setup. No Nix. No build toolchain. Skr8tr ships as a set of pre-built, patchelf-clean binaries for Linux x86-64 or ARM64. Drop them on any host, set executable, done. PQC signing keys are generated on your machine — they never leave it.

2

Write a six-line manifest for each service

app api-server exec /opt/myapp/bin/api args --port 8080 port 8080 replicas 3 restart on-failure

No YAML. No CRDs. No Helm. The .skr8tr manifest is human-readable and expressive — secrets, volumes, health checks, and autoscale rules in the same file.

3

One command deploys across your entire fleet

$ skr8tr --key ~/.skr8tr/signing.sec up api-server.skr8tr   submitting api-server... ok   scheduled → node-1 (10.10.0.11) — 3 replicas launching   registered in Tower — ingress routing active

Every deploy command is ML-DSA-65 signed before it touches the Conductor. No bearer tokens. No passwords. Your signing key stays on your machine.

4

The ingress routes traffic instantly — no config required

# Services register themselves in Tower on launch. # The ingress resolves them on every request — no nginx config, no reload. $ skr8tr tower   api-server 10.10.0.11:8080 ttl=28s replicas=3   frontend 10.10.0.12:4200 ttl=25s replicas=1   product-svc 10.10.0.12:8001 ttl=27s replicas=1

SHA-256 mini-blockchain chains every SUBMIT, EVICT, ROLLOUT, EXEC, and AUTH_FAIL. AES-256-GCM at-rest encryption. skr8tr audit-verify walks the full chain in seconds. Tamper-evident by design — any modification breaks the chain.

📡 RFC 5424 Syslog / SIEM Export

Every audit event forwarded in real time to your SIEM over TLS — Splunk, QRadar, Microsoft Sentinel, Datadog, Graylog, or any RFC 5424-compliant receiver. Covers HIPAA § 164.312(b) centralised log collection and PCI DSS 10.5.3.

✓ INCLUDED IN PLATFORM

🔐 SSO / OIDC Identity Bridge

Federate operator identity with Okta, Google Workspace, or any OIDC-compliant provider. OAuth2 flow → IdP validates identity → bridge issues a short-lived ML-DSA-65 session credential. Group claims map directly to Skr8tr RBAC namespaces.

✓ INCLUDED IN PLATFORM

🏛️ Multi-Tenant Conductor

Namespace isolation so team A cannot see or touch team B's workloads. Per-namespace resource quotas: max replicas, max RAM, max nodes, max submit rate. Real-time enforcement — over-quota SUBMIT is rejected before it touches a node.

✓ INCLUDED IN PLATFORM

📈 Auto-Scaling

CPU, memory, queue depth, and arbitrary Prometheus metric triggers. Cron-based predictive scaling for known traffic patterns. AND/OR composite rules. Min/max replica bounds enforced at the Conductor. No metrics-server, no HPA objects.

✓ INCLUDED IN PLATFORM

⚡ HTTP/2 Ingress

Full HTTP/2 with ALPN negotiation on the ingress — multiplexed streams, header compression, gRPC proxying. Compiled into skr8tr_ingress with TLS termination. The OSS ingress is HTTP/1.1; HTTP/2 is enterprise.
